## 0. 狙い（なぜこのUIにするか）

- ターゲットは**茶道初心者〜お稽古中の女性**。やさしく・明るく・上品に、所作を一手ずつ追える。
- これまでのダーク（墨×金）版は**“静かな本格”**。本テンプレは**“やわらかい品”**＝白×淡桜で女性に寄せる。
- 実シーン画像（畳・道具）は温かみがあるので、**白い余白の額縁**に納めると映える。
- 提供価値：① 自分目線で迷わない ② 縦書き明朝の和の格 ③ やさしいピクトグラムで親しみ。

### 配色（和の女性的パレット）
| 用途 | 名 | HEX |

| 背景（基調） | 生成り | `#FBF7F4` |

### 余白・形・影
- 角丸：カード `16px` / ボタン `999px`（ピル）/ 画像額 `14px`
- 余白基準：`8 / 16 / 24 / 40 / 64`（広めを基本、1手画面は上下 `64`）
- 影：`0 12px 40px rgba(58,51,46,.08)`（淡く高く＝贅沢感）
- 罫：`1px` 淡金 or `#EFE6DF`

### 縦書き／横書きの使い分け（ルール）
- **縦書き（`writing-mode: vertical-rl`）**：
  - 点前名「運び点前」、章題「一、清め」、**手番（漢数字 一〜二十八）**、ヒーローのコンセプト一文
  - 数字は**縦中横**（`text-combine-upright`）で潰さない
- **横書き**：所作の説明本文・急所・道具名・ボタン・進捗
- 1画面に縦横を混在させる時は、**縦書き＝象徴／横書き＝情報**と役割を必ず分ける

## 4. アイコン／ピクトグラム方針（女性的ライン）

- **線画（ライン）アイコン**：線幅 `1.5px`、端は丸（`round`）、角丸。色は紅梅 or 墨。
- **道具ピクト**：釜・茶碗・茶筅・棗・柄杓・帛紗…を**極細ラインの正面シルエット**で。お点前の進行マーカーや道具名チップに添える。
- **UIアイコン**：次へ/前へ＝細い矢羽、音＝小さな鈴、章＝花の小印、メニュー＝三本線に小花。
- **あしらい**：桜・梅の**小さな単弁**を区切りや選択中マークに（多用しない、一点投入）。
- 形式：SVG（線画）。後の3D/イラストとトーンを揃える。

### (A) ヒーロー
- 白背景に実シーン1枚（風炉釜）を右に大きく、左に**縦書きの点前名＋コンセプト一文**。
- 例コピー：「一服を、自分の手で。」／淡金の細罫＋桜の小印。

### (B) インデックス（分岐）
- **流儀 × 季節（炉/風炉）× 男女**を、白カード＋淡桜ホバーで選ぶ。
- 現在公開：**表千家／風炉（夏）／女点前／薄茶・運び**。他はカードを淡色で「準備中」。
- カードに道具ラインアイコン＋季節の花（風炉＝撫子、炉＝椿）。

### (C) 手順プレイヤー（中核）
```

│ ［縦書き 点前名］   ① 清め ② 点て ③ 仕舞い（タブ）   音♪ │
├──────────┬───────────────────────────────────┤
│ 手番      │   ┌── 実シーン画像（白い額・16:9）──┐    │
│ 一        │   │                                 │    │
│ 二 ●      │   └─────────────────────────────────┘    │
│ 三        │   ［道具ピクト・チップ］                  │
│ …(縦)     │   横書き：所作の説明／急所／女点前メモ   │
├──────────┴───────────────────────────────────┤
│  ← 前へ      ●━━━━━━━○ 進捗      次へ →   12/29  │
```

- 中央：**白い額縁**に実シーン（クロスフェード）。下に道具ピクトのチップ。
- 右/下：横書きで説明・急所・女点前差分。
- 最下部：**フィルムストリップ**（全手サムネ、スワイプで頭出し）。

## 7. モーション・余白の作法
- イージングは**ゆっくり**（`cubic-bezier(.22,.61,.36,1)`、`.45〜.6s`）。弾みすぎない＝所作の静けさ。
- 画面遷移は**クロスフェード**主体。道具の登場は将来のパーツ版で“すっ”と。
- 余白は**怖いくらい広く**。1手1画面、情報を詰めない。

## 11. UIルール（運用・2026-06-18 追記）

> 実装で守る決まり。リサーチ詳細は `UI_RESEARCH.md`。

1. **解説は隠せる（イメージ優先モード）**：各画面に「解説を隠す」操作を必ず置く。
   - 上部「解」アイコン＝解説の表示/非表示トグル。カード右上「解説を隠す ✕」でも非表示化。
   - 状態は `localStorage`（`sado_explain`）で記憶。イメージだけで進めたい人を妨げない。
2. **豆知識を添える（学べる脇知識）**：各手に茶道の豆知識を1つ表示（`data/tidbits.js`／抹茶色の左罫）。
   - 千利休・歴史・季節・茶杓の銘・茶碗・道具・作法・流儀・精神の各カテゴリ。捏造禁止・確実な事実のみ。
3. **役割分離**：縦書き＝象徴（手番・章題）／横書き＝情報（解説・道具・豆知識）。混ぜない。
4. **主軸インターフェース＝クックモード型＋進捗パス**：1手＝1画面・大画像・スワイプ送り。全体は一本道で現在地表示。
5. **格納＝ドロワー**：種目ツリー等は普段畳んで画像に集中、必要時のみ展開（スマホover／タブレットpush）。
6. **3カラムは拡張表示専用**：PC/タブレットのみ（左ナビ｜中央所作｜右注記）。スマホでは必ずドロワーに畳む。
7. **データ層と描画の分離を維持**：`steps.js`／`tidbits.js` は無改修で着せ替えできる前提（将来の3D化に備える）。
